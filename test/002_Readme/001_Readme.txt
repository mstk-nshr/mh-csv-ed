・AI生成素材を使用する際は
  「適切な権利がある素材」で学習されたモデルやサービス以外で
  生成したAI生成素材の使用を禁止します。
  以下はこの基準です。
  (※予告なく基準を変更する場合があります！)
  ・Adobeの「Adobe Firefly」や、
    Stable Diffusion用の「Mitsua Diffusion」モデルのような
    「パブリックドメインやCC0の素材、
    許諾ありの素材のみで学習されたAI生成素材」(オプトイン制)
    のAIで生成した素材のみ使用可
  ・Stable Diffusion用の基本モデルを含むほぼ全てのモデル、NovelAI、Midjourney系、
    Dall-E(Bing、Windows Copilot、ChatGPT)などの、学習元データが明示されていない、
    または「モデル/サービス提供者が権利を持たない画像」で
    学習されたAIで生成した素材は使用禁止

  ゲーム制作ソフトとしての配布を、クレジット表記を行うことを条件に許可します。
  ただし、素材単体での販売や自作発言は禁止とします。
  また、ゲーム制作ソフトとして配布する場合は、以下の条件を満たすことを必須とします。
  ・完全無料であること
  ・処理内容などがオープンソースで公開されていること
  ・ソースコードの二次配布、改造、および改造物の配布を許可すること
  ・広告を一切表示しないこと
  ・課金要素を設けないこと
  ・見返りを求めないこと
  ・同梱素材の規約を守ること
    (特に、二次配布禁止の素材は同梱不可)

・敵キャラに蘇生スキルを使用させても敵が蘇生しない
  (基本システム(ウディタ同梱とAdvancedも含める)では、敵が戦闘不能になった時点で
   その敵キャラが戦闘スロットから消滅するため正常な仕様です)
  敵キャラに蘇生スキルを使用させる際は
  敵専用のスキルから敵キャラの蘇生コモンを呼び出してください

・UDB「状態設定1」ではUDB「汎用ステータス変化1」の「自動付与状態」が機能しない
  ・自動付与状態による付与する状態の無限ループを防ぐための仕様です

・戦闘不能時に蘇生する効果のある状態にかかってる時、以下の条件を全て満たすと
  蘇生時の効果音とポップアップとログが二重に再生される
  ・ダメージで戦闘不能になった
  ・状態付与で戦闘不能になった
  (表示上の問題だけで、システム面には問題ないので仕様としています)

・戦闘不能時に蘇生する効果のある状態にかかった敵が戦闘不能時に蘇生しない
  (敵の戦闘不能時にスロットが消滅するので、これによって起こるバグを封じるため
   敵に対しては戦闘不能時に蘇生する効果のある状態は機能しません)

・能力値やダメージ計算の過程で、±999999を超えた場合、
  自分の属性威力、相手の属性耐性、スキルの庇護ダメージ防御値、状態の全ダメージ耐性等で
  ダメージが減少されると本来の計算結果より値が少なくなる
  (±20億を超えると、正常な結果が得られなくなり、
   ゲームバランスに深刻な悪影響を及ぼすため)
  (スキルのダメージ量が±999999を超えないようにバランス調整してください)

・「[変更可]戦闘時ｷｬﾗ戦闘不能Ev」および、敵グループごとのキャラ戦闘不能コモンに
  メンバーの増減が挿入されている場合、エラーが出る事がある
  (該当のイベントコマンド内でメンバーの増減コマンドを使用しないでください！)

・テストプレイ時、F11で再読み込みができない
  (ゲーム開始時やロード時にUDB項目のファイル名を置き換えており
   再読み込みを行うとファイルが見つからないエラーが発生するため)
  (デバッグコマンドの「F11再読み込み」コマンドを利用してください)

・有料版ウディタで画面サイズを「[Pro]自由サイズ[3倍]」にした場合
  マップ暗転や戦闘開始時の演出が変になる
  ・ウディタの仕様です

・有料版ウディタで「[P]使用済ﾌｧｲﾙｺﾋﾟｰ」を使用した場合、以下のファイルがコピーされない
  ・複数のページのあるイベントにて、それぞれのページに設定されている画像が違ったり
    設定されていなかったりする場合、コピーされない画像がある
  ・マップチップはコピーされない
  ・マップデータもコピーされない
  ・「BasicData」フォルダ内のファイルもコピーされない
  ・「_RTP_Sys」フォルダ内のファイルもコピーされない
  これらのファイルは自力でコピーする必要があります
