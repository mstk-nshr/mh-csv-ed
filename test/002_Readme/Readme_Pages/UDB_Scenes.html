■シーン関連設定

●ﾉﾝﾌｨｰﾙﾄﾞﾀﾞﾝｼﾞｮﾝ設定1

  • タグ:カスタム
    • 上級者向けの設定をするための機能です。
      カスタムタグについては別紙を参照ください。
  • ダンジョン説明/推奨Lv
    • ダンジョンの説明文と推奨Lvを設定します。
      出撃するかの選択肢で表示されます。
  • ランダムEv設定1/ランダムEvシード値1
    • ダンジョン内で発生するランダムイベントを設定します。
      シード値を1以上にすると乱数を固定化し、常に決まったイベントが発生するようになります。
  • 全フロア数
    • ダンジョンの全フロア数です。
  • 開始位置/開始向き
    • ノンフィールドダンジョンで使用する位置を設定します。
  • 撤退位置/撤退向き
    • 引き返す/撤退/全滅時に移動する位置を設定します。
  • 完走位置/完走向き
    • 完走時に移動する位置を設定します。
  • 移動時コモンEv
    • 移動時に実行されるコモンイベントを指定します。
  • クリア済Ev引数
    • そのダンジョンをクリア済の場合、最終イベントと固定イベントとランダムイベントに設定したコモンのセルフ3番に1を代入するか0を代入するかを設定します。
  • 出撃前EvコモンID1
    • 出撃前に実行されるコモンイベントです。
      該当のコモンイベントのコモンセルフ0番にはダンジョンID、3番には出撃済フラグが代入されます。
      出撃済フラグは一度でも出撃すると、完走したか撤退したかに関係なくONになります。
  • クリア後EvコモンID1
    • 完走時に実行されるコモンイベントです。
      該当のコモンイベントのコモンセルフ0番にはダンジョンID、3番にはクリア済フラグが代入されます。
      クリア済フラグは一度でも完走するとONになります。
  • 最終EvコモンID1
    • 最後のマスで実行されるコモンイベントを設定します。
      該当のコモンイベントのコモンセルフ0番にはダンジョンID、1番にはフロア数、2番には引数、3番にはクリア済フラグが代入されます。
      コモンセルフ10番を「-1」にすると全滅とみなします。
      クリア済フラグは一度でも完走するとONになります。
  • 固定EvコモンID1/固定Ev引数1/固定Evフロア数1
    • 固定イベントとして実行されるコモンイベントとその引数とそれが発生するフロア数を設定します。
      該当のコモンイベントのコモンセルフ0番にはダンジョンID、1番にはフロア数、2番には引数、3番にはクリア済フラグが代入されます。
      コモンセルフ10番を「-1」にすると全滅とみなします。
      クリア済フラグは一度でも完走するとONになります。

●ﾉﾝﾌｨｰﾙﾄﾞ確率設定1

  • イベント1コモンEv1/イベント1引数1/イベント1優先度1
    • ランダムイベントとして実行されるコモンイベントとその引数とその優先度を設定します。
      優先度が高いほどそのイベントが抽選されやすくなります。
      該当のコモンイベントのコモンセルフ0番にはダンジョンID、1番にはフロア数、2番には引数、3番にはクリア済フラグが代入されます。
      コモンセルフ10番を「-1」にすると全滅とみなします。
      クリア済フラグは一度でも完走するとONになります。

●クエスト分類

クエストの分類名を指定します。

●クエスト設定

  • クエスト名
    • クエスト名です。
  • タグ:カスタム
    • 上級者向けの設定をするための機能です。
      カスタムタグについては別紙を参照ください。
  • クエスト説明[進行前～完了]
    • 受注前、進行中、完了時のクエストの説明文を設定します。
  • 分類
    • そのクエストのクエスト画面での分類です。
      UDB「クエスト分類」からデータIDで指定します。
  • クエスト種別
    • クエストの種別をモンスター討伐、アイテム納品、その他の中から設定します。
  • 撃破対象敵キャラID1～5、必要撃破数1～5
    • 撃破対象の敵キャラについて設定します。
      クエスト種別がモンスター討伐でのみ有効な項目です。
  • 必要物品種別、アイテム1～3、個数1～5
    • クエストの完了に必要なアイテムについて設定します。
      クエスト種別がアイテム納品でのみ有効な項目です。
  • 報酬種別、アイテム1～3、個数1～5
    • 報酬として受け取るアイテムについて設定します。
  • 報酬金額
    • クエストの完了時に報酬として受け取る金額です。

●実績設定

  • 実績名
    • 実績名です。
  • 達成時の説明文～条件不明時の説明文
    • 達成時、条件判明時、条件不明時の説明文です。
  • アイコン、アイコン拡大率
    • アイコンをUDB「アイコン画像設定」からデータIDで指定します。
  • ごほうび画像、ごほうび画像拡大率
    • 決定キーを押した時にごほうび画像が表示されます。
  • 達成率条件[0-100]
    • 設定された達成率を達成すると自動的に獲得します。
  • 最初から判明済みか
    • 最初から条件が判明するかを設定します。
  • タグ:カスタム
    • 上級者向けの設定をするための機能です。
      カスタムタグについては別紙を参照ください。

  • 文章
    • ヘルプブックで表示されます。
  • 分類

      UDB「ヘルプ分類」からデータIDで指定します。
  • 閲覧可能フラグ
    • 常に閲覧できるか、入手判定に依存するかを設定します。
  • タグ:カスタム
    • 上級者向けの設定をするための機能です。
      カスタムタグについては別紙を参照ください。

●あらすじ設定

  • 項目名
    • 表示名です。
  • あらすじ内容
    • あらすじ画面で表示される文章です。
  • シナリオID条件
    • あらすじの閲覧条件をシナリオIDで判定させます。
  • 変数1～3名、変数1～3条件
    • あらすじの閲覧条件を通常変数の値で判定させます。

●あらすじ分類2

  • 表示名
    • 表示する分類名です。
  • チェックリスト
    • 使わない
      • 進行度を表示しません。
    • カウンターを使う
      • 進行度を「●●●○○」といった具合に表示します。
    • [○%]を使う
      • 進行度をパーセント形式で表示します。
    • [○/○]を使う
      • 進行度を分数形式で表示します。
  • [1:カウンター]未達成
    • 「チェックリスト」がカウンターの場合、未達成として表示するマークです。
      • 例:○
  • [1:カウンター]達成
    • 「チェックリスト」がカウンターの場合、達成済として表示するマークです。
      • 例:●
  • ［1:%,分数］ゲージ
    • 「チェックリスト」が分数の場合、ゲージとして表示するかを設定します。
  • ［使う］完了時
    • このあらすじの完了時、この項目の1個下の項目に設定された文字列に置き換えるかを設定します。
  • [使う]失敗時に表示
    • このあらすじの失敗時、この項目に設定された文字列に置き換えるかを設定します。
  • 達成状況が未開放の項目
    • 表示しないか、「—-」「???」に置き換えるかを設定します。
  • 各状況が変更された時
    • 「変化させる変数」の値を置き換えるかを設定します。
  • 変化させる変数
    • 変更対象となる通常変数の値です。
  • 形式
    • =最大の完了ID
      • そのカテゴリで進行状況が「完了」の項目のデータ番号の中で
        一番大きい物を100倍した値を代入します
        該当するものが無かった場合、「-100」を代入
    • =最小の進行中ID
      • そのカテゴリで進行状況が「進行中」の項目のデータ番号の中で
        一番小さいものを100倍したものを代入します
        該当するものがなかった場合、「最大の完了ID*100＋その報告数」を代入
    • =最小の進行中ID+最大報告
      • そのカテゴリで進行状況が「進行中」の項目のデータ番号の中で
        一番小さいものを100倍したものに一番大きい「達成」の報告番号を
        足した値を代入します
        該当するものがなかった場合、「最大の完了ID*100＋その報告数」を代入
    • =最小の進行中ID+最小目標
      • そのカテゴリで進行状況が「進行中」の項目のデータ番号の中で
        一番小さいものを100倍したものに一番小さい「目標」の報告番号を
        足した値を代入します
        該当するものがなかった場合、「最小の進行中ID*100+最大報告」を代入
    • =最後に変更したID
      • コモンイベントを呼び出すときに使ったあらすじID*100を代入
    • =最後に変更したID+最大報告
      • 最小の進行中～と同じような感じ
    • =最後に変更したID+最小目標
      • 最小の進行中～と同じような感じ
    • =進行中のID数
      • 進行中の項目数を代入
    • =進行中以上のID数
      • 進行中以上の項目数を代入
    • =完了済みのID数
      • 完了済みの項目数を代入
  • 進行中/完了ﾘｽﾄに表示しない
    • 進行中や完了済のリストに表示するかしないかを設定します。
      操作説明などの形式の分類はこれに設定してください。

●あらすじ設定2

  • 表示名
    • このあらすじの表示名です。
  • カテゴリ
    • このあらすじのカテゴリ名です。
  • 背景画像
    • このあらすじの背景に表示する画像です。
  • 目標数
    • このあらすじの目標数です。
  • ①～⑮目標/┗報告
    • 目標には未報告時の文章を、報告には報告済の文章を入れてください。
  • 次のあらすじID
    • 次のあらすじIDです。
  • 周回時初期化するか
    • 周回時にあらすじを未達成にするかを設定します。
  • 常時/ﾌﾞﾗｳｻﾞ版/有料版初期閲覧可能[ﾋﾞｯﾄ]/常時初期目標設定[ﾋﾞｯﾄ]
    • 最初から各目標を報告済/設定済の状態にします。
      目標番号に応じたビット数は以下の通りです。
      「1」と入れると目標1が、「6」と入れると目標2と目標3が報告済/設定済となります
      • 目標1
        • ビット数1
      • 目標2
        • ビット数2
      • 目標3
        • ビット数4
      • 目標4
        • ビット数8
      • 目標5
        • ビット数16
      • 目標6
        • ビット数32
      • 目標7
        • ビット数64
      • 目標8
        • ビット数128
      • 目標9
        • ビット数256
      • 目標10
        • ビット数512
      • 目標11
        • ビット数1024
      • 目標12
        • ビット数2048
      • 目標13
        • ビット数4096
      • 目標14
        • ビット数8192
      • 目標15
        • ビット数16384
  • ｸｴｽﾄ討伐対象ｴﾈﾐｰﾘｽﾄ
    • 「DataName:」を文字列の先頭に設定するとIDではなくデータ名で設定します。
      「,」または改行で複数設定できます。
      敵キャラのID、またはデータ名で設定します。
      • 例:
        • DataName:(改行)大ネズミ:3,洞窟ネズミ:1,…
  • ｸｴｽﾄ納品対象ｱｲﾃﾑﾘｽﾄ
    • 「DataName:」を文字列の先頭に設定するとIDではなくデータ名で設定します。
      「,」または改行で複数設定できます。
      アイテムのID、またはデータ名で設定します。
      • 例:
        • DataName:(改行)サンドイッチ:3,チョココロネ:1,…
  • ｸｴｽﾄ報酬ｱｲﾃﾑﾘｽﾄ/ﾘﾌﾟﾚｲ報酬ｱｲﾃﾑﾘｽﾄ
    • 「DataName:」を文字列の先頭に設定するとIDではなくデータ名で設定します。
      「,」または改行で複数設定できます。
      アイテムのID、またはデータ名で設定します。
      1回目の報酬は「ｸｴｽﾄ報酬ｱｲﾃﾑﾘｽﾄ」で、2回目の報酬は「ｸｴｽﾄﾘﾌﾟﾚｲ報酬ｱｲﾃﾑﾘｽﾄ」で設定します。
      「ｸｴｽﾄﾘﾌﾟﾚｲ報酬ｱｲﾃﾑﾘｽﾄ」が空欄の場合、2回目の報酬は1回目の報酬と同じになります。
      • 例:
        • DataName:(改行)ポーション:2,エリクサー:1,…
  • ｸｴｽﾄ報酬金額/ﾘﾌﾟﾚｲ報酬金額
    • クエスト完了時の報酬を設定します。
      1回目の報酬は「ｸｴｽﾄ報酬金額」で、2回目の報酬は「ｸｴｽﾄﾘﾌﾟﾚｲ報酬金額」で設定します。
      「ｸｴｽﾄﾘﾌﾟﾚｲ報酬金額」が「-1」以下の場合、2回目の報酬は1回目の報酬と同じになります。
  • ｸｴｽﾄ受注条件目標[番号]
    • クエストの受注条件として設定する目標番号です。
  • ｸｴｽﾄ受注時目標設定[ﾋﾞｯﾄ]
    • クエスト受注時にその目標番号が表示されます。
      ビット値で設定します。
      ビット値と目標番号の関係は「常時/ﾌﾞﾗｳｻﾞ版/有料版初期閲覧可能[ﾋﾞｯﾄ]」を参照してください。
  • ｸｴｽﾄ敵討伐条件目標[ﾋﾞｯﾄ]
    • クエストの敵討伐完了時にその目標番号が報告されます。
      ビット値で設定します。
      ビット値と目標番号の関係は「常時/ﾌﾞﾗｳｻﾞ版/有料版初期閲覧可能[ﾋﾞｯﾄ]」を参照してください。
  • ｸｴｽﾄ納品条件目標[ﾋﾞｯﾄ]
    • クエストのアイテム納品完了時にその目標番号が報告されます。
      ビット値で設定します。
      ビット値と目標番号の関係は「常時/ﾌﾞﾗｳｻﾞ版/有料版初期閲覧可能[ﾋﾞｯﾄ]」を参照してください。
  • ｸｴｽﾄ完了条件目標[番号]
    • クエストの完了条件として設定する目標番号です。
  • 実績ごほうび画像ﾌｧｲﾙ/拡大率
    • ごほうび画像が設定されたあらすじ2をリストから開いた際に決定キーを押すと設定した画像を開けます。
      その拡大率も設定します。
  • 実績達成率条件[0-100]
    • 任意のあらすじ2を実績として完了させた際、実績完成率に応じて自動的に完了します。

●権利表記

  • 項目名
    • 「画像素材」や「音楽素材」などの項目名です。
      「@」を記述するとそれ以降の文字を表示しません。
  • 人物名txtファイル
    • 「txt」形式のファイルを指定してください。
      ファイルの内容がそのまま表示されます。
      テキストファイルの文字コードを「UTF-8(BOM付き)」にしてください！
      「ANSI」「UTF-8」では文字化けします！
      テキストファイルの中身には人物名を入力してください。
  • 最終項目
    • これを1に設定すると、これを最後の項目として、これより後の項目を表示しません。
      上級者向けの項目です！
      通常は0に設定する事をおすすめします！

●スタッフロール1

  • 項目名
    • 「@」を記述するとそれ以降の文字を表示しません。
      0番目のデータにはゲームのタイトル名を入れてください。
      1番目～最後のデータより1個前のデータには「画像素材」や「コモンイベント素材」等の役割名
      最後のデータには「遊んでくれてありがとう」
  • 人物名
    • 0番目のデータには「制作スタッフ」を入れてください。
      1番目～最後のデータより1個前のデータには人物の名前かサイト名
      最後のデータには「By.(作者名)」を入れてください。
  • URL
    • サイトのURLを入れてください。
      空欄でも大丈夫です。
      0番目のデータと最後のデータには無効です。
  • 備考
    • 任意の文章を入れてください。
      空欄でも大丈夫です。
      0番目のデータと最後のデータには無効です。
  • 背景
    • 0番のデータに指定するとスタッフロール全体の背景を指定できます。
      1番以降のデータで「背景」を指定しても無効です。
  • 一枚絵
    • 一枚絵です。
      0番目と最後のデータに指定するとテキストの代わりに画像を表示します。
  • 最終項目
    • これを1に設定すると、これを最後の項目として、これより後の項目を表示しません。
      上級者向けの項目です！
      通常は0に設定する事をおすすめします！
