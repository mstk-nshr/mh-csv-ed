■主人公/敵キャラ/召喚獣関連設定🦁♥🐍

●主人公設定1

  • ユーザーDBの0番「主人公設定1」で主人公のステータスを設定するように変更しました。
    これにより、バージョンアップによる主人公のステータスの変更に対応しました。
    なお、ロード毎に主人公のステータスは更新されます。
    (変更があった場合は現在のHP、MP、レベルや経験値は維持されその他のパラメーターは変更されます。)
    また、イベントで覚えたスキルやパッシブはステータスを更新しても保持されます。
    ※なお、可変DBの0番「×主人公ｽﾃｰﾀｽ保存」を設定してもゲーム開始時にユーザーDB0番の値が代入されるため、意味はありません。
  • ステータスの更新に対応しているコモンイベントは以下の通りです。
    • コモンイベント「○主人公情報の変更」
    • コモンイベント「○主人公ﾏｯﾌﾟ歩行画像変更」
    • コモンイベント「○主人公ﾊﾞﾄﾙ歩行画像変更」
    • コモンイベント「○主人公顔画像変更」
    • コモンイベント「○主人公立ち絵画像変更」
    • コモンイベント「○能力値増減」
    • コモンイベント「○経験値・Lv増減」
    • コモンイベント「○極意強化増減」
    • コモンイベント「○使用スキル増減」
    • コモンイベント「○パッシブスキル増減」
    • コモンイベント「○スキル直接セット」
    • コモンイベント「○装備アイテム1の変更」
    • コモンイベント「○装備アイテム2の変更」
    • コモンイベント「○装備アイテム3の変更」
    • コモンイベント「○主人公装備全部外す」
    • コモンイベント「○主人公装備変更許可設定」
  • ステータスの更新に対応していないコモンイベントは以下の通りです。
    コモンイベント「[変更可]主人公変更分適用」に、イベント中の下記の操作を入れてください。
    • コモンイベント「○戦闘コマンドの変更」
  • デフォルトの経験値獲得システムの場合、次のレベルの必要経験値の計算方法は以下の通りです。A値、B値、C値が大きいほど、必要経験値が増えます。ただし、Lv×Lvにおいて、Lvの上限は100で、100×100=10000です。(A値とB値とC値を高くしすぎた時、Lv100～999で累計経験値をインフレさせないためLv999の累計経験値の上限が2億になるよう、そのように調整させていただきました。
    • 「Lv×Lv×A値+Lv×B値+C値」
    • 例:レベル1→2の場合、A値=5、B値=8、C値=2
      • 「1×1×5+1×8+2」=「5+8+2」=「15」
    • 例:レベル49→50の場合、A値=5、B値=8、C値=2
      • 「49×49×5+49×8+2」=「12005+392+2」=「12399」
    • 例:レベル101→102の場合、A値=5、B値=8、C値=2
      • 「100×100×5+101×8+2」=「50000+808+2」=「50810」
    • 例:レベル500→501の場合、A値=5、B値=8、C値=2
      • 「100×100×5+500×8+2」=「50000+4000+2」=「54002」
    • 例:レベル998→999の場合、A値=5、B値=8、C値=2
      • 「100×100×5+999×8+2」=「50000+7992+2」=「57994」
    • 例:レベル998→999の場合、A値=20、B値=25、C値=100
      • 「100×100×20+999×25+100」=「200000+24975+100」=「225075」
    • レベル999の時点での累計経験値、A値=20、B値=25、C値=100
      • 「198929325」
  • 「歩行画像[バトル用]」で使用する画像の規格は以下の通りです。1モーションあたり、3コマにしてください。
    320x240の画面サイズを基準に、1コマ縦横32ピクセルの画像を推奨します。
    モーション
    待機突き逃走
    移動振り勝利
    詠唱撃ちピンチ
    防御汎用発動状態異常
    ダメージ魔法発動睡眠
    回避アイテム使用戦闘不能

  • キャラ名/肩書き/キャラ説明
    • キャラクターの基本的な情報を設定します。
  • 歩行（顔）画像[ﾏｯﾌﾟ（ﾊﾞﾄﾙ）（ﾊﾞﾄﾙ用拡大背景）用]/立ち絵[ｽﾃｰﾀｽ/カットイン用]
    • グラフィックを各種UDB、SDBからデータIDで指定します。
      カットイン用の立ち絵が設定されていない場合はステータス用の立ち絵をカットインに使用します。
  • 初期ｽﾃｰﾀｽ番号
    • 無料高機能RPGシステムMr.Hでは、「汎用初期ｽﾃｰﾀｽ」から能力値を設定します。
      その参照IDです。
  • レベル参照ID
    • 0以上
      • 他の主人公とレベルを同期します。
    • <個別>
      • 他の主人公とレベルを同期しません。
    • <ｼﾅﾘｵごとに変化>
      • シナリオごとに同期する主人公IDを変更します。
  • 初期Lv/限界Lv/限界Lvを指定する？
    • Lvの設定を行います。
      コモンイベント側でLv上限を設定するか、UDB側でLv上限を設定するかを設定します。
      コモンイベント側で設定する場合、コモンイベント「[変更可]主人公限界ﾚﾍﾞﾙ更新」で設定します。
  • 基本装備ポイント/[成長]装備ポイント増加
    • 装備重量制で、装備するためのポイントについての設定を行います。
      装備重量制のみ有効な項目です。
  • 基本極意強化値/[成長]極意強化値増加/１Lv
    • 極意によるスキル習得に使用するためのポイントについての設定を行います。
  • 基本最大スキル選択値/[成長]スキル選択値増加/1Lv
    • スキル選択制を採用する際、選び取ることのできるスキルのコストを許す数を設定します。
      スキル選択制のみ有効な項目です。
  • 最大アイテム所持数
    • 持てるアイテムの最大数を設定します。
      アイテム個別制のみ有効な項目です。
  • 最大ｽｷﾙ選択数
    • スキル選択制を採用する際、選び取ることのできるスキルのスロット数を設定します。
  • 装備欄1～15初期装備
    • ゲーム開始時点で装備している装備品を指定します。
  • 装備可能タイプ1～10
    • 装備可能なアイテムの種別を指定します。
  • 装備条件ﾀｸﾞ
    • 該当タグが設定された装備の装備可能判定を変動させます。
      改行で複数のタグを設定できます。
      無料高機能RPGシステムMr.Hでは、「汎用条件設定1」と「条件ﾀｸﾞ設定1」で使用されます。
  • 戦闘コマンド
    • 戦闘中に使用できるコマンドの一覧をデータIDで指定します。
  • 習得極意強化（スキル/パッシブ）
    • 習得する極意、スキル、パッシブの一覧を設定します。
  • ジャマー設定
    • 無料高機能RPGシステムMr.Hでは、「汎用ジャマー設定1」からジャマーの有無を設定します。
      その参照IDです。
  • 必要経験値ABC
    • 必要経験値を設定します・
      AにはLvの2乗が、BにはLvがかけられます。
      Cは固定値です。
  • 初期所属職業
    • デフォルトで所属している職業を指定します。
  • 初期職業ﾚﾍﾞﾙ
    • ゲーム開始時に職業レベルを設定します。
      「(職業ID:職業Lv)」の文字列で設定します。
      「,」で複数の職業レベルを設定できます。
      • 例:「0:1,7:2」
        • その主人公はゲーム開始時の時点で0番の職業レベルが1、7番の職業レベルが2になります。
      • 「DataName:」を先頭に入れた場合は以下を入力します
        • 「(職業名):(職業レベル),…」
          「,」で複数設定できます。
  • クラスチェンジ可能カテゴリ
    • その主人公が転職可能な職業をその属性のみに限定します。
      空欄で全ての職業に転職できます。
  • クラスロックカテゴリ
    • ゲーム開始時にその属性の職業をロックします。
      ロックされた属性の職業は職業を変更したり、職業レベルを変更する事はできません。
      改行で複数の職業の属性をロックできます。
  • 転職禁止職業
    • その主人公は設定された職業に転職できません。
      職業IDで設定します、「,」で区切って複数の職業を設定します。
      • 例:「5,7,8」
        • その主人公は5、7、8番の職業に転職できなくなる
      • 「DataName:」を先頭に入れた場合は以下を入力します
        • 「(職業名),…」
          「,」で複数設定できます。
  • AIで自動行動する？
    • 手動で行動するか、設定されたデフォルトAIで行動するか、
      作戦型AI(習得したスキルの中から自動で選択)に基づいて行動するかを設定します。
  • 対応デフォルトAI
    • 無料高機能RPGシステムMr.Hでは、「ﾃﾞﾌｫﾙﾄ行動AI」からデフォルトAIの行動内容を設定します。
      その参照IDです。
      「AIで自動行動する？」が「[1]AIで自動行動[ﾃﾞﾌｫﾙﾄAI]」の時のみ有効な項目です。
  • デフォルトAIデータ数
    • 対応ﾃﾞﾌｫﾙﾄAIからいくつ分のデータを使用するかを設定します。
      0以下の場合は1個となります。
      「AIで自動行動する？」が「[1]AIで自動行動[ﾃﾞﾌｫﾙﾄAI]」の時のみ有効な項目です。
  • ローテーションAIループ数
    • デフォルトAIのローテーション条件のループ数を設定します。
      0以下の場合は使用しません。
      「AIで自動行動する？」が「[1]AIで自動行動[ﾃﾞﾌｫﾙﾄAI]」の時のみ有効な項目です。
  • 変更禁止設定1
    • 装備変更を禁止するか、作戦変更を禁止するか、職業変更を禁止するかを設定します。
      主に、一時的に仲間になるキャラ用に設定します。
  • 周回時ステータス初期化する？
    • 強くてニューゲーム時にステータスを初期化するか設定します。
  • 一時的な味方？
    • デバッグコマンドのレベルアップの対象から除外し、メニューでHPやMP等を非表示にするかを設定します。
      主に、一時的に仲間になるキャラ用に設定します。
  • タグ:カスタム
    • 上級者向けの設定をするための機能です。
      カスタムタグについては別紙を参照ください。
  • ●●時台詞（入店/退店/購入/売却/購入キャンセル/売却キャンセル）
    • 主人公をお店の店員にしたときに表示される台詞を指定します。
  • 対象選択時台詞
    • 対象にカーソルを合わせたとき、表示される台詞を指定します

●職業設定1

  • 職業名
    • その職業の名前です。
  • 職業名[短]
    • メニュー画面でキャラの肩書きの代わりに表示される文字列です。
  • 説明文
    • 「転職画面」や「職業レベルアップ画面」で表示されます。
  • 最大職業レベル
    • その職業のレベル上限です。
  • 最大HP増減[%]～敏捷性増減[%]、ｶｽﾀﾑ能力値1増減[%]～ｶｽﾀﾑ能力値5増減[%]
    • 無職の時の能力値に乗算されます。
  • 命中率増減[%]～ｸﾘﾃｨｶﾙ増減[%]
    • 無職の時の能力値に加算されます。
  • ｽｷﾙﾀｸﾞ
    • 該当タグが設定されたスキルの発動や命中、ダメージを変動させます。
      改行で複数のタグを設定できます。
      無料高機能RPGシステムMr.Hでは、「汎用条件設定1」と「条件ﾀｸﾞ設定1」で使用されます。
  • 最大HP増減[1Lv成長]～ｶｽﾀﾑ能力値5増減[1Lv成長]
    • その職業レベルに応じて無職の時の能力値に加算されます。
      上昇量は
      「(設定値÷100)×(現在の職業Lv-職業Lvの下限)」
      です。
  • 職業についてなくても補正
    • 職業についていない状態でも、その職業レベルに応じて能力値を補正するかを設定します。
  • 装備可能タイプ1～8
    • その職業が装備できる物を指定するものです。
  • 装備条件ﾀｸﾞ
    • 該当のタグが設定された武器/防具の装備可能判定を変動します。
      改行で複数のタグを設定できます。
      無料高機能RPGシステムMr.Hでは、「汎用条件設定1」と「条件ﾀｸﾞ設定1」で使用されます。
  • 最大アイテム所持数増加
    • アイテムを持てる数を増やします。
      アイテム個別制のみ有効な項目です。
  • 戦闘コマンド
    • 戦闘中に追加するコマンドの一覧をデータIDで指定します。
  • 習得極意強化（スキル/パッシブ）
    • 習得する極意、スキル、パッシブの一覧を設定します。
  • 専用極意強化/スキル/パッシブ継承
    • 該当の職業専用のスキル/パッシブをその職業で使用できます。
      ただし、習得レベルが「2001」以上のスキル/パッシブは継承できません。
      • 例:「ウォリアー」の「専用スキル継承」に「1,2」を入れると
        「ウォリアー」に転職した際に「ファイター」「モンク」の専用スキルが使用できる。
      • 「DataName:」を先頭に入れた場合は以下を入力します
        • 「(職業名),…」
          「,」で複数設定できます。
  • スキル習得レベル参照職業
    • 極意強化/スキル/パッシブの習得レベルに関して、他の職業のレベルも参照します。
      「DataName:」を先頭に入れた場合は職業のデータ名で、そうでない場合は職業IDで設定します。
      「,」で複数設定できます。
  • 他職業Lv条件ｽｷﾙ
    • 他の職業のLvに応じて、スキル/パッシブを追加で習得させます。
      「DataName:」を先頭に入れた場合はデータ名で、そうでない場合はデータIDで設定します。
      「,」で複数設定できます。
      以下の形式で入力します。
      • (スキル種別):(習得スキルリスト):(職業ID);(職業Lv):(職業ID);(職業Lv)
        • (スキル種別)の部分に以下の文字列を入れます
          • 使用スキル
            • 使用スキルを追加で覚えます。
          • パッシブ
            • パッシブを追加で覚えます。
        • (習得スキルリスト)の部分に以下を入れます
          • (スキル種別)の部分に使用スキルを入れた場合は
            (習得使用スキルリストID)を入れる
          • (スキル種別)の部分にパッシブを入れた場合は
            (習得パッシブリストID)を入れる
  • 必要職業ID1～5、必要職業Lv1～5
    • 「必要職業IDX」が「必要職業レベルX」以上の時に転職条件を満たします。
      転職条件を満たさない時は転職できません。
  • 転職禁止フラグ
    • その職業はポイント制において、ポイントの振り分けを可能にしつつ、転職のみができなくなります。
  • クラスカテゴリ
    • その職業の属性を設定します。
      改行は使用できず、1つの属性しか設定できません。
      属性を設定すると、クラスチェンジ可能カテゴリを持ってない主人公はその職業に転職できません。
      また、同じ属性の他の職業のレベルの合計が制限されます。
      また、ロックされた属性はその職業のレベルを変更できません。
  • サブ職業フラグ
    • メイン
      • 無制限でポイントに割り振る事ができます。
    • サブ
      • そのクラスカテゴリ内のメイン職業にポイントを割り振ってないとポイントを割り振る事ができません
        • 例:ファイターがメイン職業でキャスターがサブ職業の場合
          • ファイターにポイントを割り振ってないとキャスターにポイントを割り振れない
            ファイターにポイントを割り振ってるとキャスターにポイントを割り振る事ができる
  • 転職可能主人公
    • 該当の職業はその主人公しか転職できません。
      • 例:「ブレイバー」のこの文字列項目に「0」を入れるとその職業はIDが「0」の主人公しか転職できなくなる。
      • 「DataName:」を先頭に入れた場合は以下を入力します
        • 「(主人公名),…」
          「,」で複数設定できます。
  • 職業必要経験値、職業経験値増加[固定値]、職業経験値増加[割合]
    • 必要職業経験値を設定します。
  • 最初から転職可能か
    • ゲーム開始時から転職できるかを設定します。
  • タグ:カスタム
    • 上級者向けの設定をするための機能です。
      カスタムタグについては別紙を参照ください。

●汎用初期ｽﾃｰﾀｽ1

  • 名前
    • データ名です。
  • タグ:カスタム
    • 上級者向けの設定をするための機能です。
      カスタムタグについては別紙を参照ください。
  • 盗む/難易無効/ﾒﾀﾙ/MP∞
    • 盗むスキルを無効化するか、難易度による補正を無効化するか、メタルボディにするか、MPを消費させなくするかを設定します。
      • メタルボディにした場合、受けるダメージ量は「1 + スキルや武器のメタル系追加ダメージ量」となります。ただし、クリティカルを受けた時は「通常のダメージ量 + スキルや武器のメタル系追加ダメージ量」となります。
      敵キャラのみ有効な項目です。
  • 最初からスキル反射するか
    • 常に、物理系や魔法系のスキルを反射するかを設定します。
      敵キャラのみ有効な項目です。
  • 属性耐性1～4
    • 無料高機能RPGシステムMr.Hでは、「属性耐性設定」からキャラの属性耐性を設定します。
      その参照IDです。

●敵ドロップアイテム設定1

  • 名前
    • データ名です。
  • タグ:カスタム
    • 上級者向けの設定をするための機能です。
      カスタムタグについては別紙を参照ください。
  • アイテム変化リスト1
    • 敵がアイテム変化スキルを受けた時に変化するアイテムとその確率を文字列で設定します。
      空欄の場合、アイテム変化種別/アイテム変化アイテム1～3/アイテム変化率[％]の項目を参照します。
      「DataName:」を文字列の先頭に入れるとデータIDではなくデータ名で設定します。
      「,」や改行で複数設定できます。
      「(データ名):(ドロップ率(％)),…」という書き方で設定します。
      • 例:
        • 「DataName:(改行)エーテル:50,ポーション:100」と設定した場合
          • 瀕死状態の場合、50％の確率でエーテルに、エーテルに変化しなかった場合は100％の確率でポーションに変化します。
  • 盗むアイテムリスト1
    • 敵が盗むスキルを受けた時に入手するアイテムとその確率を文字列で設定します。
      空欄の場合、盗めるアイテムとその確率はドロップアイテムと同じになります。
      「DataName:」を文字列の先頭に入れるとデータIDではなくデータ名で設定します。
      「,」や改行で複数設定できます。
      「(データ名):(ドロップ率(1/2000)),…」という書き方で設定します。
      • 例:
        • 「DataName:(改行)ポーション:1200,エーテル:600」と設定した場合
          • 60％の確率でポーションを、30％の確率でエーテルを盗めます。
  • ドロップアイテムリスト1
    • 敵を倒した時に入手するアイテムとその確率を文字列で設定します。
      空欄の場合、ドロップアイテム種別1～3/入手アイテム1～3_1～3/入手率1～3[1/2000]の項目を参照します。
      「DataName:」を文字列の先頭に入れるとデータIDではなくデータ名で設定します。
      「,」や改行で複数設定できます。
      「(データ名):(ドロップ率(1/2000)),…」という書き方で設定します。
      • 例:
        • 「DataName:(改行)ポーション:600,エーテル:300」と設定した場合
          • 30％の確率でポーションを、15％の確率でエーテルを落とします。
  • ドロップアイテム種別1～3、入手アイテム1～3_1～3
    • 指定したアイテム1～3を落とします。
  • 入手率1～3[1/2000]
    • ドロップ率は1/2000の確率で指定してください。
      • 例:「入手率1～3[1/2000]」を「125」と指定すると6.25％の確率で落とす。
  • アイテム変化種別、アイテム変化アイテム1～3
    • 敵がアイテム変化スキルを受けた時、指定したアイテム1～3に変化します。
  • アイテム変化率[％]
    • 敵がアイテム変化スキルを受けた時の変化率を指定します。
      HPが低いほどアイテム変化が成功しやすくなります。
      「100」と指定した場合、HP全快で成功率は10％となります。

●敵キャラ画像設定

  • 名前
    • データ名です。
  • 画像ファイル
    • 「_MV」「_MZ」が末尾にない
      • サイドビュー戦闘でも通常の敵画像として表示されます。
    • 「CharaChip」(参照先変更可)フォルダが参照されている
      • サイドビュー戦闘時は4/8方向規格の歩行画像として扱われます。
        ただし、フロントビューの場合は通常の敵画像として表示されます。
    • 「_MV」「_MZ」が末尾にある
      • サイドビュー戦闘時はMV規格の歩行画像として扱われます。
        ただし、フロントビューの場合は通常の敵画像として表示されます。
  • 異常時画像
    • 状態異常時に表示する顔画像です。
      顔画像の表示でのみ有効です。
  • ピンチ時画像
    • 状態異常にかかっておらずHPが25％以下の時に表示する顔画像です。
      顔画像の表示でのみ有効です。
  • 戦闘不能時画像
    • 戦闘不能時に表示する顔画像です。
      顔画像の表示でのみ有効です。
  • 画像拡大率
    • 画像の拡大率です。
  • 画像横拡大率反転
    • 画像を横に反転させます。
  • 不透明度-255-0
    • 画像の不透明度を変更します。
  • 追加Yシフト量
    • 画像のY座標をシフトさせます。
  • カラー補正R/G/B±100
    • 画像のRGBの度合いを変更します。
      RGB全てを「100」にしても真っ白になりません。
  • 色相0-360
    • 画像の色相を変更します。
  • 彩度±255
    • 画像の彩度を変更します。
      「-255」でモノクロになります。
  • 明度±255
    • 画像の明度を変更します。
      「-255」で黒に「255」で真っ白になります。
  • モノトーン
    • 画像を特定の色一色にします。
      色相が「0」で赤一色になります。
  • 階調反転
    • 画像の色を反転させます。
  • ぼかし
    • 画像をぼかします。
  • 上下浮遊/拡大縮小
    • 非選択時、上下浮遊するか、選択時、拡大縮小するかを設定します。
  • ATBX/Y座標補正値
    • ATBバーの表示位置をシフトします。

●召喚獣設定1

  • 召喚獣の名前
    • 召喚獣の名前です。
      • 「%1」を含める
        • 召喚獣の発動者の名前に置き換えます。
  • 確認時表示枠色
    • その召喚獣の確認時の表示枠の色です。
  • 重複するか
    • 重複して呼び出せるかを指定します。
  • 味方か敵か
    • その召喚獣が敵か味方かを設定します。
  • 発動者の行動制限確認
    • 発動者が行動不能でも行動するかを設定します。
  • 誰に発生させるか
    • 召喚獣の対象を設定します。
  • 初期ｽﾃｰﾀｽ番号
    • 無料高機能RPGシステムMr.Hでは、「汎用初期ｽﾃｰﾀｽ」から能力値を設定します。
      その参照IDです。
  • Lv[-1:発動者と同じ]
    • Lvの設定を行います。
      「-1」以下を設定すると、主人公のレベルと同じになります。
  • 行動回数
    • その敵キャラの行動回数です。
  • 行動回数増加率
    • 値は「(追加で確率判定を行う回数×1000)+(確定で増加する回数×100)+(行動回数が1回増加する確率)」で指定してください。
      • 例:「3250」と指定すると確率で行動回数が最低で2回、最大で6回増加する
  • 対応デフォルトAI
    • 無料高機能RPGシステムMr.Hでは、「ﾃﾞﾌｫﾙﾄ行動AI」からデフォルトAIの行動内容を設定します。
      その参照IDです。
  • デフォルトAIデータ数
    • 対応ﾃﾞﾌｫﾙﾄAIからいくつ分のデータを使用するかを設定します。
      0以下の場合は1個となります。
  • ローテーションAIループ数
    • デフォルトAIのローテーション条件のループ数を設定します。
      0以下の場合は使用しません。
  • 最短ターン数[0以下:∞]、最大ターン数
    • ターン数について設定します。
      「最短ターン数[0以下:∞]」を0以下にするとターン数は無限となります。
  • 発生時のメッセージ～消滅時のメッセージ
    • 発生時、重複時、消滅時のメッセージです。
      • 「%1」を含める
        • 召喚獣の発動者の名前に置き換えます。
      • 「%2」を含める
        • 召喚獣の名前に置き換えます。
  • タグ:カスタム
    • 上級者向けの設定をするための機能です。
      カスタムタグについては別紙を参照ください。
